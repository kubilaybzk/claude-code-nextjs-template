# Accessibility (a11y) — Minimum Rules

## Icon-Only Buttons

Buttons containing only an icon require an `aria-label`.

```tsx
// ✗ Forbidden
<Button variant="ghost" size="icon">
  <TrashIcon />
</Button>

// ✓ Correct
<Button variant="ghost" size="icon" aria-label="Delete company">
  <TrashIcon />
</Button>
```

## Screen Reader Only Text

Use the `sr-only` class to provide context for screen readers.

```tsx
<Button>
  <PlusIcon />
  <span className="sr-only">Add new company</span>
</Button>
```

## Form Fields

Every input must be associated with a label. shadcn's `<FormLabel>` handles this automatically.

```tsx
// ✓ FormLabel auto-links htmlFor
<FormItem>
  <FormLabel>Company Name</FormLabel>
  <FormControl>
    <Input placeholder="e.g. Acme Ltd." {...field} />
  </FormControl>
  <FormMessage />
</FormItem>
```

## Color Must Not Be the Only Indicator

Error, warning, and success states must not rely on color alone — add an icon or text.

```tsx
// ✗ Color only
<Badge className="bg-destructive">Error</Badge>

// ✓ Icon + color
<Badge variant="destructive">
  <AlertCircleIcon className="size-3" />
  Error
</Badge>
```
